BellSoft publishes Liberica JDK and JRE as ready-to-run container images for
cloud and on-premise deployment. They come in three families — pick the one
that matches your platform and security needs.

## Hardened images (recommended)

`bellsoft/hardened-liberica-runtime-container` — near-zero-CVE **nonroot** and
**distroless** Liberica JDK images, built on [Alpaquita Linux](/bellsoft-docs/alpaquita/stream/general/install-guide/).
Best default for new projects: smallest attack surface and production-ready
security out of the box. See [Hardened Liberica JDK Images](hardened/).

## Runtime container

`bellsoft/liberica-runtime-container` — Alpaquita-based images for general use
when you need a shell and standard tooling. Three image types:

- **jdk** — Liberica JDK Lite, optimized for cloud.
- **jdk-all** — full JDK with `jlink` for building custom runtimes.
- **jre** — runtime only, no development tools.

See [Using the container images](usage/) to pull and run them, and the
[Liberica Runtime Container](tags/) page to build a valid tag.

## Distribution-based images

Liberica JDK/JRE on established base distributions — Alpine, Debian, CentOS,
Rocky Linux, and Windows Server Core. Use these when your platform standardizes
on a specific base OS. See [Distribution-based Liberica Images](distributions/).
